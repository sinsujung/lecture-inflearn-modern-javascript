[React의 특징]

    컴포넌트를 기반으로 UI를 표현한다

    컴포넌트(Component)

        우리말로 "구성요소"라는 뜻

        화면을 구성하는 요소, UI를 구성하는 요소를 말함

    Header, Main, Footer로 나누어 중복코드를 줄인다

    화면 업데이트 구현이 쉽다

        업데이트

            사용자의 행동(클릭, 드래그)에 따라 웹 페이지가 스스로 모습을 바꿔 상호작용 하는 것

    선언형 프로그래밍

        과정은 생략하고 목적만 간결히 명시하는 방법

        마치 식당에서 주문하는 것과 유사

    명령형 프로그래밍

        목적을 이루기 위한 모든 일련의 과정을 설명하는 방식

        마치 진상 손님처럼 주문하는 방식

    선언형 프로그래밍(React)는 목적만 깔끔하게 명시하여 코드가 간결하고 명령형 프로그래밍(JS)는 모든 과정을 하나 하나 다 설명하여 코드가 길고 복잡하다

    React를 사용하면 컴포넌트의 상태를 저장하는 State 변수가 있는데 State 변수의 값에 따라 랜더링 결과를 다르게 할 수 있다

    업데이트를 위한 복잡한 동작을 직접 정의할 필요 없이 특정 변수의 값을 바꾸는 것 만으로도 화면을 업데이트 시킬 수 있다

    화면 업데이트가 빠르게 처리된다

    화면 업데이트를 쉽게 구현할 수 있으면서 동시에 빠르게 처리해준다.


    브라우저 랜더링 과정(Critical Rendering Path)
        HTML -> DOM(요소들의 위치, 배치, 모양에 관한 모든 정보)

        CSS -> CSSOM(요소들의 스타일과 관련된 모든 정보)

        DOM, CSSOM 변경 후 -> Render Tree(웹 페이지 설계도) -> Layout(요소 배치 작업) -> Painting(실제 화면에 그려내는 과정)

    Virtual DOM
        DOM을 자바스크립트 객체로 흉내낸 것으로 일종의 복제판

        React는 업데이트가 발생하면 실제 DOM을 수정하기 전에 이 가상의 복제판 DOM에 먼저 반영한다.
